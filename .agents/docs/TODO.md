# Project To-Dos

Items extracted from JOURNAL.md during good-sleep consolidation. Each item should be resolved or removed once addressed.

## Open Items

- [x] Support parameterized queries in the transpilation pipeline. - *source: 2026-03-29: User-driven expansion and sqlparser 0.61 upgrade* - **Resolved**: sqlparser-rs handles placeholders (`$1`, `:name`, `?`) as `Value::Placeholder` and they pass through transpilation unchanged. No papera-specific code needed.
- [x] Rewrite Redshift 3-arg `dateadd` and `datediff` forms into DuckDB-compatible interval arithmetic instead of simple renaming. - *source: 2026-03-29: User-driven expansion and sqlparser 0.61 upgrade* - **Done 2026-03-29**: `DATEADD(part, n, date)` now emits `date + INTERVAL 'n' PART`; `DATEDIFF(part, start, end)` emits `date_diff('part', start, end)` with quoted datepart.
- [x] Handle nested Trino `ROW` types instead of relying on the current flat modifier string parser. - *source: 2026-03-29: Initial Implementation* - **Closed as upstream limitation**: sqlparser-rs 0.61 `parse_optional_type_modifiers()` cannot handle nested parentheses in type positions. All dialects fail equally. Flat `ROW(a INT, b VARCHAR)` works; nested `ROW(x BIGINT, y ROW(i DOUBLE, j DOUBLE))` does not. Tracked as a parser constraint in ARCHITECTURE.md.
- [x] Add a mapping for `ROW FORMAT SERDE 'class'` in Hive external table handling. - *source: 2026-03-29: Initial Implementation* - **Done 2026-03-29**: `determine_reader_function` now falls back to SerDe class name matching when no `STORED AS` is present. Supports ParquetHiveSerDe, OrcSerde, JsonSerDe, OpenCSVSerde, LazySimpleSerDe.
- [x] Implement format string conversion where source dialect format specifiers differ from DuckDB `strftime` and `strptime`. - *source: 2026-03-29: Initial Implementation* - **Done 2026-03-29**: Added `src/transforms/format_strings.rs` with converters for Redshift (PostgreSQL-style) and Trino (Java DateTimeFormatter-style) to DuckDB strftime format. Applied to `format_datetime`, `date_parse` (Trino) and `to_char`, `to_date`, `to_timestamp` (Redshift). Formats already in strftime `%`-style are passed through unchanged.
- [x] Decide whether to support Redshift `COPY` and, if so, map the supported subset to DuckDB behavior. - *source: 2026-03-29: Initial Implementation* - **Done 2026-03-29**: Added `CopyBehavior` option (default `Error`, opt-in `MapToInsert`). Rewrites `COPY table FROM 's3://...'` to `INSERT INTO table SELECT * FROM read_parquet/read_csv/read_json(...)`. Maps Redshift options (DELIMITER, IGNOREHEADER, GZIP, etc.) to DuckDB reader params. Silently drops IAM_ROLE, MANIFEST, COMPUPDATE, etc. Redshift default pipe delimiter is preserved.
- [x] Decide whether to map Redshift system views such as `svv_*` and `stl_*` to DuckDB-accessible equivalents. - *source: 2026-03-29: Initial Implementation* - **Closed as won't-fix**: The mappable subset (SVV_TABLE_INFO, SVV_COLUMNS, SVV_ALL_SCHEMAS) has incompatible column schemas, so queries selecting specific columns would still break. STL_* (query logs) and STV_* (runtime snapshots) have no DuckDB equivalents. Users querying system views are doing Redshift-specific ops and should rewrite manually.
